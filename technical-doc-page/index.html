Programming Environment

Setting up a programming environment is neccessary before even starting on writing your own program. Doing so will allow you to effectively write, test and run your code.

Data Types

By definition, a data type is an attribute of data which tells the compiler/interpreter how your program intends to use the data. For example, if we use the integer data type, our program knows that we are dealing with whole numbers, and that we can perform certain operations on this type such as basic arithmetic operations.

Variables

Variables in programming are names used to hold one or more values.

Keywords

Keywords are predefined, reserved words used in programming that have special meaning to the compiler. For example, in the Java programming language, using the keyword void in a function declaration means that the function returns nothing.

Logical and Arithmetical Operators

Logical operators deal with truth, or boolean, values. For example, the three main logical operators you will encounter in programming is AND, OR and NOT. The symbol used to represent each of these operators will depend on the programming language, but its purpose remains the same.

If-Else Conditions

Programs may need to execute different code depending on certain conditions. For example, a calculator program would need to perform addition if a user enters the addition symbol, and subtraction if the user enters the subtraction symbol. This is often handled using If-Else conditions.

If condition 1 occurs { do these things } else { do these other things }
Loops

Sometimes your program might need to repeat a certain action indefinitely or until a certain condition is met. For example, consider a simple counting program: you prompt the user to enter a number to add to the current count, and the program ends once the count reaches, or is above 100.

We could write code that prompts the user for a number, add the number to the count, and then write more code to prompt the user for another number and so on. However, how many times do we need to do this? We can see here that there would be a lot of duplicated code.

Prompt user for number, add number to count, check count Prompt user for number, add number to count, check count repeat... (how many times?)

We can use a loop instead so that our code doesn't repeat itself. In Java, this would look something like this: while (count <= 100) { prompt user for number, add number to count, check count }

The program will simply repeat the count increment until the count does not meet the condition of the loop (count is less than or equal to 100).
